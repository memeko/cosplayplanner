# Cosplay Planner

Веб-приложение для ведения косплана с авторизацией, прогрессом работы и базой фестивалей.

## Что уже реализовано

- Регистрация, вход, выход, хранение пользователей в SQLite.
- Отдельный профиль участника: смена ника, email, пароля, основного города.
- Вкладка `Косплан`:
  - список карточек + поиск;
  - создание/редактирование карточки;
  - экспорт всех карточек пользователя в CSV;
  - поддержка shared-карточек (если вас указали по нику).
- Вкладка `In Progress`:
  - добавление карточки из косплана;
  - чеклист (добавить/отметить/удалить).
- Вкладка `Фестивали`:
  - карточки фестивалей;
  - фильтры по городу, номинации, сокосплееру;
  - лавандовая подсветка фестивалей из основного города пользователя;
  - сводка ближайших 30 дней (скрывается при активных фильтрах);
  - экспорт в календарь `.ics`;
  - оповещения от других пользователей;
  - отдельная подсветка фестивалей, пришедших из shared-карточек.

## Важные сценарии

- В карточке косплана можно указать сокосплеера не только текстом, но и по нику пользователя.
- Если ник найден, у этого пользователя автоматически создаётся shared-карточка и появляется оповещение во вкладке фестивалей.
- По каждой карточке считается ориентировочная стоимость проекта (сумма всех ценовых полей).

## Импорт с cosplay2.ru

Импорт берёт данные с `https://cosplay2.ru/` из публичного JSON-LD (`application/ld+json`):

- название,
- URL,
- город,
- дата события,
- дедлайн подачи (если явно найден в тексте описания).

Повторный импорт аккуратно дополняет существующие записи.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Открыть: `http://127.0.0.1:8000`

## Health endpoints

- `GET /healthz` — liveness.
- `GET /readyz` — readiness (проверка SQL-запроса к БД).

## Переменные окружения

См. `.env.example`.

- `SECRET_KEY`
- `SESSION_HTTPS_ONLY`
- `TRUSTED_HOSTS`
- `DATABASE_URL`
- `PORT`
- `PROJECT_NAME_FILE`

По умолчанию приложение использует `/data/cosplay.db`, если каталог `/data` существует и доступен на запись, иначе `./cosplay.db`.
`SECRET_KEY` обязателен: небезопасный fallback в коде отключён.

## Имя проекта

Имя проекта хранится в `PROJECT_NAME.txt`.
