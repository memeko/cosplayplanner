# Cosplay Planner

Веб-приложение для ведения косплана с авторизацией, прогрессом работы и базой фестивалей.

## Что уже реализовано

- Регистрация, вход, выход, хранение пользователей в SQLite.
- Отдельный профиль участника: смена ника, email, пароля, основного города и секретного кода для Telegram-бота.
- Вкладка `Косплан`:
  - список карточек + поиск;
  - создание/редактирование карточки;
  - экспорт всех карточек пользователя в CSV;
  - поддержка shared-карточек (если вас указали по нику).
- Вкладка `In Progress`:
  - добавление карточки из косплана;
  - чеклист (добавить/отметить/удалить).
- Вкладка `Фестивали`:
  - карточки фестивалей;
  - фильтры по городу, номинации, сокосплееру;
  - лавандовая подсветка фестивалей из основного города пользователя;
  - сводка ближайших 30 дней (скрывается при активных фильтрах);
  - экспорт в календарь `.ics`;
  - оповещения от других пользователей;
  - отдельная подсветка фестивалей, пришедших из shared-карточек.
- Вкладка `Контент-план`:
  - редактор постов для Telegram;
  - автопубликация в Telegram, VK и Pinterest;
  - ручной трекинг публикаций в RedNote/Xiaohongshu, если нужен отдельный учёт этой площадки без публичного API.

## Важные сценарии

- В карточке косплана можно указать сокосплеера не только текстом, но и по нику пользователя.
- Если ник найден, у этого пользователя автоматически создаётся shared-карточка и появляется оповещение во вкладке фестивалей.
- По каждой карточке считается ориентировочная стоимость проекта (сумма всех ценовых полей).

## Импорт с cosplay2.ru

Импорт берёт данные с `https://cosplay2.ru/` из публичного JSON-LD (`application/ld+json`):

- название,
- URL,
- город,
- дата события,
- дедлайн подачи (если явно найден в тексте описания).

Повторный импорт аккуратно дополняет существующие записи.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Открыть: `http://127.0.0.1:8000`

## Health endpoints

- `GET /healthz` — liveness.
- `GET /readyz` — readiness (проверка SQL-запроса к БД).

## Переменные окружения

См. `.env.example`.

- `SECRET_KEY`
- `SESSION_HTTPS_ONLY`
- `TRUSTED_HOSTS`
- `DATABASE_URL`
- `PORT`
- `PROJECT_NAME_FILE`
- `TELEGRAM_BOT_TOKEN` (необязательно, включает Telegram-бота)
- `TELEGRAM_BOT_ENABLED` (`1`/`0`, по умолчанию `1` при наличии токена)
- `TELEGRAM_POLL_TIMEOUT` (секунды long-polling, по умолчанию `20`)
- `TELEGRAM_LOOP_SLEEP` (пауза цикла воркера, по умолчанию `2`)
- `APP_BASE_URL` (базовый URL приложения для ссылок восстановления пароля)
- `PASSWORD_RESET_TOKEN_MINUTES` (TTL ссылки восстановления, по умолчанию `30`)
- `SMTP_HOST`
- `SMTP_PORT` (обычно `587`)
- `SMTP_USER`
- `SMTP_PASSWORD`
- `SMTP_FROM_EMAIL`
- `SMTP_USE_TLS` (`1`/`0`)
- `SMTP_USE_SSL` (`1`/`0`)

По умолчанию приложение использует `/data/cosplay.db`, если каталог `/data` существует и доступен на запись, иначе `./cosplay.db`.
`SECRET_KEY` обязателен: небезопасный fallback в коде отключён.

## Telegram-бот уведомлений

После задания `TELEGRAM_BOT_TOKEN` приложение поднимает текстового Telegram-бота внутри того же процесса FastAPI.

Сценарий авторизации в боте:

1. Отправить `/start` или `/login`.
2. Ввести ник на сайте (username или cosplay_nick).
3. Ввести секретный код для ТГ-бота из профиля на сайте.

После привязки бот отправляет пользователю уведомления по событиям:

- `Курлык` (личные сообщения);
- добавление/обновление shared-карточки по нику;
- назначение задания в карточке `В работе`;
- новый комментарий в карточке мастера пользователя.

Команда `/logout` отвязывает Telegram от аккаунта сайта.

## Восстановление пароля

На странице входа есть ссылка «Забыли пароль?». Пользователь вводит email и получает одноразовую ссылку для смены пароля.

- токен восстановления хранится в БД только в виде SHA-256 хеша;
- ссылка ограничена по времени (`PASSWORD_RESET_TOKEN_MINUTES`);
- токен одноразовый (после успешной смены пароля помечается использованным).
