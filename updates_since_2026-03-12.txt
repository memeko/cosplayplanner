2) Все возможности программы (актуально на 19.03.2026)
-------------------------------------------------------

A. Аккаунт и безопасность
1. Регистрация, вход/выход, авторизация через сессию.
2. Профиль: ник на сайте, ник косплеера, email, пароль, основной город (можно несколько через запятую), день рождения.
3. Поиск пользователей по никнейму сайта и по нику косплеера.
4. Страница Политики обработки персональных данных + уведомление о согласии при регистрации.
5. Приветственное модальное окно после регистрации.
6. Для спец-аккаунта brfox_cosplay (angenzel@gmail.com): статистика пользователей/карточек/городов.
7. SECRET_KEY берется из переменной окружения (без небезопасного хардкода).

B. Главная (Новости)
1. Лента оповещений (новые карточки по нику, задания, конфликты, репетиции и т.д.).
2. Кнопки «Отметить прочитанным» и «Очистить список оповещений».
3. Кнопка «Не учитывать» на конкретных уведомлениях.
4. Блок «Дни рождения» (участники на текущей неделе).
5. Блок «Инфоповоды на ближайшую неделю».
6. Блок «Дни рождения персонажей на сегодня».
7. Блок «Отправить голубя»: персональное сообщение пользователю по нику.
8. Входящий «голубь» приходит всплывающим popup-уведомлением «🕊 Курлык!».

C. Косплан
1. Список карточек + поиск.
2. Создание/редактирование/удаление карточек.
3. Детальная карточка показывает только заполненные поля; ссылки кликабельны.
4. Экспорт всех карточек пользователя в CSV.
5. Полная структура карточки:
   - Основное: персонаж, фандом, AU.
   - Костюм: покупка/пошив, подварианты, сроки, цены, валюта, комментарий.
   - Обувь, линзы, парик, крафт (со своими сценариями и ценами).
   - Доп.ссылки для костюма/крафта: ссылка + цена + комментарий.
   - Фотосет: дата, фотографы, студии, разбивка стоимости, комментарий, чеклист реквизита.
   - Выступление: трек/ссылка, видеофон, сценарий, светосценарий, длительность, репетиционная точка, стоимость репетиций.
   - Референсы и референсы поз/кадров (в т.ч. Pinterest embed).
   - Примечания.
6. В поле цен есть поддержка «цена неизвестна» -> в итоге «УТОЧНЯЕТСЯ».
7. Автоподсчет ориентировочной стоимости проекта с учетом всех ценовых блоков и валюты.
8. Сокосплееры: ввод с подсказками по базе, добавление нескольких участников.
9. Руководитель проекта: привязка по нику/алиасу.
10. Связи с другими карточками (с указанием названия карточки и автора).
11. Планируемые фестивали под спойлером + фильтр по городу.
12. Комментарии в карточке (ветки сообщений и ответы).
13. Блок конфликтов дат внутри карточки.
14. Если пользователь указан в карточке как сокосплеер/руководитель, он может ее редактировать.

D. В работе
1. Перенос карточки из Косплана в «В работе».
2. Чеклист: добавить/отметить/удалить.
3. Заморозка проекта (карточка выделяется и сортируется вниз).
4. Подсветка карточек с дедлайном менее 2 недель.
5. Для карточек общего доступа: блок «Задания» (ответственный, текст, статус выполнения).
6. Уведомления исполнителю о назначенных заданиях.
7. Отображение/обработка предложенных руководителем репетиций.

E. Репетиции
1. Отдельная вкладка с карточками репетиций по коспланам.
2. Автоподтягивание дедлайна из карточки проекта.
3. До 10 дат репетиций на карточку.
4. Личный проект: репетиции сразу попадают в «Мой календарь».
5. Групповой проект: статусы предложений, согласование руководителем.
6. Руководитель может массово предложить репетиции участникам.
7. Участник может принять/отклонить предложение.
8. Поддержка удаления репетиций.

F. Мои проекты
1. Карточки, где пользователь указан руководителем.
2. Комментарии руководителя в карточках участников.
3. Массовое предложение репетиций выбранным участникам.
4. История/сводка по статусам репетиций участников (принято/отклонено).

G. Мой календарь
1. Месячные списки занятости.
2. Графическая сетка календаря по месяцам.
3. Цветовая маркировка событий: фестивали, фотосеты, репетиции, личные события, мульти-дни.
4. Подсветка совпадений событий на одну дату.
5. Отображение города и сокосплееров в событиях.
6. Добавление личных событий (дата/время/название/пояснение) и удаление.

H. Фестивали
1. Личная база карточек фестивалей (создание/редактирование/удаление).
2. Поддержка многодневных фестивалей (дата начала + дата окончания).
3. Быстрая отметка «Я иду» прямо в карточке списка (без открытия формы).
4. Указание сокосплееров для фестиваля.
5. Фильтры: город, номинация, сокосплеер, только «я иду».
6. Поиск города с нормализацией и учетом вариаций/опечаток (например СПб/Санкт-Петербург).
7. Подсветка фестивалей из городов профиля (лавандовая).
8. Подсветка фестивалей ближайших крупных городов (синяя).
9. Сводка ближайших 30 дней.
10. Экспорт фестивалей в календарь (.ics).
11. Импорт фестивалей с cosplay2.ru.
12. Система анонсов: пользователь отправляет заявку, модератор одобряет/отклоняет.
13. Одобренные анонсы публикуются всем пользователям как глобальные карточки.

I. Сообщество
1. Поиск в проекты:
   - публикация карточек поиска,
   - поиск по тексту,
   - фильтр «Мои запросы»,
   - статусы («Найдено!», «Не актуально») с цветом,
   - редактирование автором.
2. Вопрос-ответ:
   - карточки вопросов,
   - комментарии,
   - статус «Вопрос решен».
3. Мастера:
   - карточки мастеров (тип, описание, галерея, прайс-лист),
   - фильтры/поиск,
   - комментарии,
   - редактирование и удаление автором.
4. Студии:
   - карточки студий (название, город, адрес, галерея, контакт, прайс-лист, теги),
   - фильтрация по городам и тегам,
   - редактирование/удаление автором.
5. Статьи:
   - тема, автор, markdown-текст, теги,
   - встраивание YouTube,
   - комментарии,
   - избранное и фильтрация по избранному,
   - поиск по названию и тегам,
   - бейдж «авторитет» при высоком интересе к материалам.

J. Медиа, БД и эксплуатация
1. Хранение данных в SQLite, миграции без перезаписи пользовательского наполнения.
2. Автоматические резервные копии БД на сервере (скрыто от пользовательского UI).
3. Аплоад изображений в галерейные блоки с серверной обработкой:
   - конвертация в WebP,
   - сжатие до 30 КБ,
   - ширина до 512 px.
4. Health endpoints: /healthz и /readyz.
5. Конфигурация под деплой на Amvera (включая persistence).
